Client Radar.

An automated sweep that finds local businesses with no web presence, enriches each lead, and drafts outreach — with a human approving every send.

in productionautomationdata pipelineinternal tool
Build something like this →
the problem

What we were handed.

Finding local businesses that need a website is hours of manual map-scrolling and note-taking — the kind of work that quietly never gets done.

And when it does get done, it's inconsistent: different notes, missed contact details, outreach written from scratch every time.

what we built

The system.

An automated sweep of open map data that finds businesses with no web presence, enriches each lead with contact details and context, and drafts tailored outreach for each one.

The system deliberately keeps a human in the loop: nothing sends without review. The machine does the research and the first draft; a person makes the call.

Sweep

Open map data scanned for businesses with no web presence.

Enrich

Contact details and context gathered per lead, automatically.

Draft

Tailored outreach written for each business — not a mail-merge blast.

Review

A human approves every message before anything sends.

the outcome

What changed.

Prospecting went from an afternoon of manual research to a reviewed queue that's waiting each morning. It's the same pattern we build for clients: boring, valuable work done by a system instead of a person.
