The demo pipeline.

Idea to live, shareable demo in minutes — every mockup gets its own voxelbox.org subdomain, TLS included, no server work per site.

Build something like this → Browse the demo shelf
the problem

What we were handed.

The best pitch for a business without a website is showing them their site — not a portfolio PDF. But standing up a hosted, shareable demo used to cost an afternoon of server work per prospect: a vhost, a certificate, DNS, deployment.

At one demo a month that's tolerable. As a sales motion, it's a bottleneck.

what we built

The system.

A publishing pipeline on our own hardware. Every demo gets its own voxelbox.org subdomain with TLS through secure tunnels — no ports opened, no per-site server setup.

One JSON feed drives the whole catalog, and the voxelbox.org homepage reads the same feed — so new work automatically appears on the front page the moment it's published.

Router

One reverse proxy fronts every demo site — adding one is configuration, not surgery.

Tunnels

TLS and public reachability without exposing the home network.

Feed

A single JSON file is the catalog, the demo shelf, and the homepage wall.

Zero-touch

Publish a demo and the homepage updates itself — no HTML edits.

the outcome

What changed.

Idea to live, shareable demo in minutes. It's the machinery behind the “Floating in production” wall on our homepage — and behind every mockup we hand a prospect.
